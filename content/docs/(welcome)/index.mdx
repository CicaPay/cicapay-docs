# Welcome to the CicaPay API Documentation

The infrastructure that enables you to **accept and send cryptocurrency payments with ease**.

At CicaPay, we build powerful tools that help `businesses`, `institutions`, `developers`, and `individuals` across **Africa** integrate crypto payments into their products and services. ## Authentication

All API requests are authenticated using a short-lived **access token** obtained from your API keys. Here is how to get set up:

<Steps>

<Step>

### Create and Verify Your Account

Sign up at [dashboard.cicapay.com](https://dashboard.cicapay.com) and complete the account verification process before you can generate API keys.

</Step>

<Step>

### Generate Your API Keys

Once verified, open your Dashboard and navigate to **Settings → API Keys**, then click **Create API Key**.

![API Keys section in the CicaPay Dashboard](/assets/dash.jpg)

You will receive two keys:

| Key | Description |
|---|---|
| **Public Key** | Identifies your account. Safe to reference in server-side code. |
| **Secret Key** | Used to request access tokens. Treat it like a password. |

<Callout type="error" title="We Cannot Recover Your Secret Key">
Your secret key is never stored as plain text on our side. **If you lose it, it is gone** — you will need to revoke the key and generate a new one. Store it in a secrets manager or environment variable immediately after creation.
</Callout>

<Callout type="warn" title="Revoke Immediately If Compromised">
If you have any doubt about the security of a key — rotate it right away from **Dashboard → API Keys → Revoke**. Do not wait.
</Callout>

</Step>

<Step>

### Get an Access Token

Use your public key and secret key to obtain a short-lived access token from our authentication endpoint:

[**`POST /auth/token` — Authentication endpoint →**](/docs/api-reference/Authentication/createAccessToken)

```http
POST /auth/token
Content-Type: application/json

{
  "public_key": "pk_live_...",
  "secret_key": "sk_live_..."
}
```

The response returns a bearer token:

```json
{
  "access_token": "eyJhbGciOi...",
  "expires_in": 3600
}
```

</Step>

<Step>

### Authenticate Your Requests

Include the access token in the `Authorization` header of every API call:

```http
Authorization: Bearer eyJhbGciOi...
```

That's it — you are now authenticated and can use all available endpoints.

</Step>

</Steps>

<Callout type="warn" title="Payout Service — IP Whitelisting Required">
To use the **Payout** service, your server IP address must be whitelisted by our team. This involves an additional security review. [Contact us](/docs/support) to initiate the process before going live with payouts.
</Callout>
